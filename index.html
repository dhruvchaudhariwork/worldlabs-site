AI has made it easy to generate a game. It still can't tell you whether the game is any good, or how to make it great.

Mission

Fun doesn't show up in a unit test. Game feel, pacing, and coherence decide whether anyone keeps playing, and they only register in the judgment of people who have shipped games and watched players bounce off them.

So we build RL environments: instrumented game worlds where models are asked to generate the hard parts. A mechanic with a twist, a level with a difficulty curve, an economy that has to hold. Every output goes in front of shipped game developers under blind, structured conditions, and their verdicts become clean eval and reward signal.

So that models and agents can build games that aren't just playable, but worth playing.

RL / Evals

Environments to train in. Evals to prove it.

RL environments

Purpose-built game environments where models generate and modify mechanics, levels, and systems under controlled conditions: instrumented, resettable, and scored. Each one is designed with shipped game developers, so the tasks measure what actually makes games work instead of what's easy to automate.

Expert evals

Inside every environment we run generation tasks judged by vetted game developers: did the model build the mechanic the prompt asked for, and is it any good? Anchored rubrics, pairwise preferences, and written critiques, delivered as verdicts for your team and as training signal for RLHF and reward models.
