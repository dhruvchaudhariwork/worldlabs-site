Core Modules

Four agents. One command center.

Each module runs autonomously but reports to a single dashboard. Human-in-the-loop approval keeps you in control.

Dispute Desk

Detects new chargebacks, compiles order evidence, fulfillment timelines, and tracking data. Drafts reason-specific responses and generates PDF evidence bundles. Pushes to Slack for one-click approval.

Auto-evidence PDF bundles Slack alerts

Support Autoresolver

Monitors your support inbox for common issues — WISMO, cancellations, address changes. Classifies intent, pulls relevant Shopify data, and drafts empathetic replies. Auto-sends simple cases; escalates edge cases with full context.

Intent classification Auto-reply Smart escalation

Inventory Alerts

Computes days-of-cover for every SKU. Flags stock-out risks (<7 days) with auto-drafted PO emails. Detects excess inventory (>45 days) and generates promo copy to move product.

How It Works

Agent workflows, step by step

Each agent follows a deterministic pipeline with LLM reasoning at key decision points. Human approval gates prevent any auto-action on sensitive tasks.

1

DisputeAgent

  • 1Trigger: Shopify dispute webhook or Gmail chargeback notification
  • 2Fetch order JSON, fulfillment events, tracking number, customer emails
  • 3Compile evidence: timeline, tracking screenshots, email transcripts, T&C
  • 4LLM drafts reason-specific dispute response using templates
  • 5Output: Markdown summary + PDF bundle + Slack approval message
2

SupportAgent

  • 1Trigger: New support email via Gmail push notification
  • 2Classify intent: WISMO, Cancel, Refund, Address Change, Other
  • 3Gather context: order data, tracking status, subscription state
  • 4LLM drafts reply using intent-specific template + Shopify data
  • 5Auto-send if confidence > threshold; else queue for approval
3

InventoryAgent

  • 1Scheduled job runs every 6 hours via cron
  • 2Compute: days_of_cover = on_hand / avg_daily_sales per SKU
  • 3<7 days: Slack alert + auto-drafted PO email to supplier
  • 4>45 days: LLM generates promo copy to move inventory
  • 5Dashboard shows all SKUs with recommended action

Implementation

8 days to a working MVP

Lean build, fast iteration. Each step delivers a testable increment.

Day 0.5

Bootstrap Project

Repo setup, environment variables, Postgres schema, initial migrations.

Day 1 — 1.5

Shopify + Gmail Connectors

OAuth flows, webhook endpoints with HMAC verification, Gmail push/polling setup.

Day 1.5 — 3.5

DisputeAgent MVP

Data fetch pipeline, LLM evidence compilation, PDF generation, Slack notification with interactive buttons.

Day 3.5 — 5.5

SupportAgent MVP

Intent classifier (gpt-4o-mini), template library per intent, confidence-gated auto-send, human approval flow.

Day 5.5 — 6.5

InventoryAgent

Days-of-cover computation, threshold alerts, promo copy generation, supplier PO drafts.

Day 6.5 — 7.5

Dashboard UI

Next.js command center with tabs, approve/edit/reject actions, audit log, real-time updates.

Day 7.5 — 8

Testing + Hardening

Edge cases, retry logic, error handling, monitoring, and logging setup.
