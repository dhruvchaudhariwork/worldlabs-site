HOW OPSFLOW WORKS

Three agents, one command center

Each module runs autonomously but reports to a single dashboard. Human-in-the-loop approval keeps you in control.

1

Ingest

Shopify webhooks and Gmail watchers detect disputes, support emails, and inventory changes in real time. Events are routed through a Redis-backed queue to the right agent.

2

Reason

Specialized AI agents classify intent, compile evidence, compute inventory metrics, and draft responses using Claude 3.5 and GPT-4o with store-specific templates.

3

Act

Auto-send low-risk replies, push drafts to Slack for approval, generate PDF evidence bundles, and alert on inventory thresholds — all logged in an audit trail.

AGENT WORKFLOWS

Step by step, fully auditable

Each agent follows a deterministic pipeline with LLM reasoning at key decision points. Human approval gates prevent auto-action on sensitive tasks.

1

DisputeAgent

  • 01Shopify dispute webhook or Gmail chargeback email triggers agent
  • 02Fetch order JSON, fulfillment events, tracking, customer comms
  • 03Compile evidence: timeline, tracking screenshots, T&C
  • 04LLM drafts reason-specific dispute response
  • 05Output: Markdown + PDF bundle + Slack approval
2

SupportAgent

  • 01New support email via Gmail push notification
  • 02Classify: WISMO, Cancel, Refund, Address Change, Other
  • 03Pull order data, tracking status, subscription state
  • 04LLM drafts reply with intent-specific template
  • 05Auto-send if confidence > threshold, else queue
3

InventoryAgent

  • 01Scheduled job runs every 6 hours via cron
  • 02Compute days_of_cover = on_hand / avg_daily_sales
  • 03<7 days: Slack alert + auto-drafted PO email
  • 04>45 days: LLM generates promo copy
  • 05Dashboard shows SKUs with recommended action
